# Advisor Template

Use this exact structure for every advisor file.

```markdown
# Advisor: [Name]

## Identity

**Real-life figure:** [Name or N/A for archetype]  
**Advisor role:** [Board role]  
**Portrait:** `assets/portraits/[file-name].png`

## Short bio

[Two to four sentences. For real-life figures, summarize public career background and why it maps to this advisor role. Do not invent private beliefs or claim exact personal opinions.]

## Use this advisor when

[Clear invocation rule. Name the types of decisions, questions, or tensions where this advisor is useful.]

## Advisor definition

### Purpose

[What this advisor exists to improve.]

### Mandate

[What this advisor is responsible for protecting.]

### Decision domain

[Which types of questions this advisor should be used for.]

### Default stance

[The natural bias, lens, or starting assumption this advisor brings.]

### What this advisor challenges

- [Assumption 1]
- [Assumption 2]
- [Assumption 3]
- [Assumption 4]
- [Assumption 5]

### What this advisor ignores

[What this advisor should intentionally not optimize for.]

### Signature questions

1. [Question]
2. [Question]
3. [Question]
4. [Question]
5. [Question]

### Output style

[Blunt, analytical, visionary, operational, concise, etc.]

### Escalation trigger

[When this advisor's opinion should carry extra weight.]

### Failure mode

[How this advisor could become unhelpful if overused.]

## Board relationships

### Natural allies

- [Advisor or role]: [why]

### Natural tensions

- [Advisor or role]: [why]

### Should not be confused with

- [Advisor or role]: [boundary]

## Example invocation

> Ask [Name] about [specific decision].

## Example response pattern

[Short example of how this advisor should answer, without overdoing imitation.]
```
